# ドメイン概要

## ドメインの全体像

poke-api-stadium のドメインは **PokeAPI から取得したポケモンデータの表現** を中心とする。  
ゲームロジック（勝敗判定等）は Application 層の usecase に置き、Domain 層は概念モデルと変換ロジックのみを担う。

---

## 主要概念

### ポケモン

- 図鑑番号（1〜1025）によって一意に識別される
- 英語名・日本語名・公式アートワーク画像を持つ
- 1〜2 種類のタイプを持つ（単タイプの場合はタイプ2なし）
- HP・こうげき・ぼうぎょ・とくこう・とくぼう・すばやさの 6 種のステータスを持つ
- 初登場した世代（第 1〜9 世代）を持つ

### タイプ

- 全 18 種類（ノーマル / ほのお / みず / でんき / くさ / こおり / かくとう / どく / じめん / ひこう / エスパー / むし / いわ / ゴースト / ドラゴン / あく / はがね / フェアリー）
- タイプごとに「相性」が定義されており、攻撃効果は無効 / 半減 / 通常 / 2倍 の 4 段階
- タイプ相性はゲームロジック（じゃんけん等）で参照する

### ステータス

| 名称       | ゲーム内の意味           |
| ---------- | ------------------------ |
| HP         | 最大体力                 |
| こうげき   | 物理攻撃の威力           |
| ぼうぎょ   | 物理攻撃への耐性         |
| とくこう   | 特殊攻撃の威力           |
| とくぼう   | 特殊攻撃への耐性         |
| すばやさ   | 行動順（素早さ）         |

### 世代

- ポケモンが初登場したゲーム世代（第 1〜9 世代）
- 世代フィルタにより、ゲーム内で使用するポケモンの範囲を絞り込む

---

## 図鑑番号の制約

PokeAPI における図鑑番号には以下の構造がある。

| 種別         | 範囲         | 件数    | 備考                           |
| ------------ | ------------ | ------- | ------------------------------ |
| 通常フォルム | 1〜1025      | 1025 体 | 世代フィルタの対象             |
| 別フォルム   | 10001〜10277 | 277 体  | 図鑑番号なし（世代フィルタ外） |

| 概念           | モデル            | ファイル                                        | 状況 |

> 凡例: ✅ 完全実装 / 🚧 一部実装 / ⏳ 未実装 / ❌ 実装不要
