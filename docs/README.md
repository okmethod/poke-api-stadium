## ドキュメント構成

```
# ストック情報（永続的な情報）
docs/
├── domain/        # ドメイン知識: ビジネスルールと実装のマッピング
├── architecture/  # アーキテクチャ: 技術的な構成・方針
└── adr/           # ADR: 設計判断の記録
```

### ドキュメント更新が必要になるタイミング

| ドキュメント      | 更新タイミング                                 |
| ----------------- | ---------------------------------------------- |
| **domain/**       | 新しいドメインルールを検討する時 or 実装した時 |
| **architecture/** | アーキテクチャを大きく変更した時               |
| **adr/**          | 重要な技術的決定をした時                       |
